# TaskFleet 用户使用指南

## 快速入门

### 系统要求
- Windows 10/11 操作系统
- 至少2GB可用内存
- 500MB可用磁盘空间
- 稳定的网络连接

### 系统架构

TaskFleet 任务执行专家系统包含三个核心组件：

1. **服务器后端** (Rust + Axum)
   - 任务管理和分发
   - 用户认证和权限控制
   - 数据统计和分析

2. **Web前端** (React + TypeScript)
   - 管理员控制面板
   - 任务创建和监控
   - 系统配置界面

3. **员工客户端** (Tauri + Rust)
   - 员工任务接收
   - 任务状态更新
   - 工作时间记录

### 快速开始

#### 管理员用户

1. **访问Web控制台**
   - 打开浏览器访问系统地址
   - 使用管理员账号登录

2. **创建任务**
   - 在任务管理页面点击"新建任务"
   - 填写任务详情和分配信息
   - 设置优先级和截止时间

3. **监控进度**
   - 查看任务完成情况
   - 实时跟踪员工工作状态

#### 员工用户

1. **启动桌面客户端**
   - 运行TaskFleet员工客户端
   - 输入登录凭证

2. **接收任务**
   - 查看分配的任务列表
   - 了解任务详情和要求

3. **执行任务**
   - 更新任务状态（开始/暂停/完成）
   - 添加工作备注
   - 上传相关文件

## 功能说明

### 任务管理

#### 创建任务
1. 登录管理员控制台
2. 进入"任务管理"页面
3. 点击"新建任务"按钮
4. 填写必要信息：
   - 任务标题和描述
   - 分配员工
   - 优先级设置
   - 截止时间

#### 任务监控
1. **监控仪表板**
   - 查看所有任务的实时状态
   - 员工工作进度跟踪
   - 系统性能指标

2. **任务状态**
   - 🟢 进行中: 员工正在执行
   - 🟡 待处理: 已分配但未开始
   - 🔴 逾期: 超过截止时间
   - ✅ 已完成: 任务完成

#### 员工管理
- **添加员工**: 创建新员工账户
- **权限设置**: 配置员工访问权限
- **工作统计**: 查看员工工作效率
- **任务分配**: 将任务分配给合适员工

### 工作流程

#### 标准任务流程
1. 管理员创建任务
2. 系统分配给相应员工
3. 员工接收任务通知
4. 员工执行任务并更新状态
5. 管理员审核任务结果
6. 任务完成或需要修改

#### 紧急任务处理
- 高优先级标记
- 即时通知推送
- 快速响应机制
- 实时状态更新

## 员工客户端操作指南

#### 任务通知

- 系统托盘提醒
- 音频提示（可配置）
- 邮件通知（可选）

#### 状态更新
1. **开始任务**
   - 点击"开始工作"按钮
   - 系统记录开始时间
   - 状态变更为"进行中"

2. **暂停任务**
   - 临时暂停工作
   - 记录暂停时间
   - 可添加暂停原因

3. **完成任务**
   - 提交工作成果
   - 上传相关文件
   - 添加完成备注

#### 工作记录
- **时间跟踪**: 自动记录工作时间
- **进度更新**: 设置任务完成百分比
- **备注添加**: 记录工作过程和问题
- **文件上传**: 上传工作相关文档

## 常见问题解答

### 登录问题

**Q: 无法登录系统怎么办？**
A: 
1. 检查网络连接是否正常
2. 确认用户名和密码正确
3. 联系管理员重置密码
4. 清除浏览器缓存后重试

**Q: 客户端连接服务器失败？**
A:
1. 检查服务器地址配置
2. 确认防火墙设置
3. 验证网络连接状态
4. 联系技术支持

### 任务问题

**Q: 收不到任务通知？**
A:
1. 检查通知设置是否开启
2. 确认客户端是否在线
3. 验证网络连接稳定性
4. 重启客户端程序

**Q: 任务状态无法更新？**
A:
1. 检查网络连接
2. 重新登录系统
3. 联系管理员确认权限
4. 查看系统错误日志

### 文件上传问题

**Q: 文件上传失败？**
A:
1. 检查文件大小限制
2. 确认文件格式支持
3. 验证网络连接稳定
4. 重试上传操作

**Q: 上传的文件无法下载？**
A:
1. 确认文件是否上传完成
2. 检查下载权限设置
3. 清除浏览器缓存
4. 使用其他浏览器尝试
